# Decision engine

Load this when the work is a choice between options. Pair with `executive-lenses.md` for material decisions and with `governance-evidence-qa.md` for confidence and sign-off language.

## Steps

1. Define the actual decision in one sentence. Vague requests ("should we do X?") often hide a more specific decision; name it.
2. Note the decision owner if known. If unknown, use a role, not a guessed name.
3. Note the deadline or urgency if stated; otherwise say it's unknown rather than inventing one.
4. Identify criteria and separate must-haves from preferences.
5. Gather the evidence actually available; note what's missing.
6. Generate genuinely viable options, including "do nothing" or "defer" where that's realistic.
7. Analyse trade-offs against the criteria.
8. Assess risk for the leading option(s).
9. Apply the sceptic lens: what would make this recommendation wrong?
10. State reversibility - can this be undone, and at what cost?
11. Recommend, if the evidence supports a recommendation.
12. State confidence (see `governance-evidence-qa.md`) and what would change it.
13. Define the next action.

Avoid false precision: don't attach a percentage or exact score to a judgement that's really a reasoned estimate.

## Output shapes

Use `assets/templates/decision-brief.md` for a short decision framed for a single owner, or `assets/templates/recommendation-memo.md` when the audience is broader or more senior. `assets/templates/executive-summary.md` covers a one-page version of either.
