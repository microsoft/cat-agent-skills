# Prospect Research & Briefing Playbook

Build a structured **prospect briefing** for a target company (and optionally a specific contact). This briefing is the input the other playbooks in this suite consume. The goal is relevance: find real, current, verifiable hooks, not a company encyclopedia.

## Workflow

1. Clarify the target: company name, market/country, and (if known) the contact person and role. Ask once if missing.
2. Research with the tools available in this harness (web search, news, the company's website, public registers, the user's CRM or workspace notes if accessible). If no research tools are available, build the briefing from what the user provides and clearly mark gaps.
3. Extract only decision-relevant findings. Every claim in the briefing must carry its source (URL or "user-provided"). Discard anything you cannot source.
4. Map findings to the sender's offering using the company profile: which triggers make this prospect relevant to what the sender sells?

## Briefing output format (always this structure)

1. **Company snapshot** — 2–3 sentences: what they do, size class, market. No filler.
2. **Triggers** (max 5, ranked) — concrete, dated events or conditions that create urgency: new leadership, expansion, funding round, regulation or deadline affecting them, certification cycles, public commitments, cost pressure signals. Each with source.
3. **Likely pains** — 2–4 hypotheses derived from the triggers, each explicitly marked as hypothesis, never stated as fact.
4. **Angle** — 1–2 sentences: which strength/proof point from the company profile matches which trigger. If nothing matches honestly, say so; a bad-fit verdict is a valid result.
5. **Contact notes** (if a person was named) — role, tenure, public statements or posts relevant to the angle, with sources. Only publicly available professional information; no private data, no speculation about the person.
6. **Open gaps** — what you could not verify and what the user should check.

## Hard rules

- Never invent facts, numbers, or events. Unsourced claims do not enter the briefing.
- Distinguish facts from hypotheses explicitly and consistently.
- Research only publicly available, professional information about individuals. No compiling of private details.
- Keep the briefing under one page; it must be scannable in 60 seconds.
- Write the briefing in the user's working language; note the target market's language for the outreach itself.
