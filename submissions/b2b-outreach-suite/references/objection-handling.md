# Objection Handling Playbook

Answer typical B2B sales objections using the pattern **empathy → bridge → open question**. Typical objections: "no budget", "we already have a vendor/consultant", "we're too small", "sounds expensive", "no time", "just send us some material".

## Workflow

1. Identify the objection type.
2. Open with one empathy sentence that shows understanding and never contradicts the prospect. Objections are information, not attacks; never be confrontational.
3. Build a bridge from the objection to a matching strength of the sender's company (reframe the perceived weakness into a strength), using only verified strengths and proof points from the company profile (e.g. funding programs, ROI methodology, compliance support, documented savings examples).
4. Attach exactly **one** open, interest-based follow-up question that keeps the conversation alive.
5. Output exactly one finished answer (no A/B variants).

## Guidelines

- Tone: empathetic, friendly, at eye level; use the formal register of the target language where customary.
- No emojis: objection replies are a serious context (unlike casual DMs).
- Include the sign-off only if the user wants a complete message; otherwise just the reply text.
- Never invent numbers, funding rates, or reference cases. Use only what the company profile supports; otherwise phrase without a concrete number.
- No em-dashes; use comma, colon, or period.
- Output format: first the finished reply, then one line "Sales note:" with a single sentence explaining why this reply works.

## Example expectations

- **"We already have a consultant/vendor."** → Show understanding, never disparage the incumbent, point to one specific gap the sender's company covers (from the profile), and ask openly how the incumbent handles that topic. Then the sales note.
- **"No budget."** → Empathy, reframe toward funding options or ROI (only with verified figures from the profile), open question about their concrete situation. Then the sales note.
- **"Just send material."** → Agree, but qualify instead of sending a standard catalog: offer one industry-relevant example and ask an open question to narrow the topic. Then the sales note.
