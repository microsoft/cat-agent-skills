# Blog Post Structure Patterns

Use these patterns to restructure a draft without inventing new claims.

## Pattern 1: Tension-led post

1. Hook: name the tension.
2. Common mistake: explain the prevailing but flawed approach.
3. Reframe: introduce the better way to think about it.
4. Practical consequences: show what changes.
5. Takeaway: land the point.

## Pattern 2: Experience-led post

1. Observation from practice.
2. Why it matters.
3. What usually gets missed.
4. What a better approach looks like.
5. What readers should do next.

## Pattern 3: Framework-led post

1. Problem the framework solves.
2. Framework at a glance.
3. How to use it.
4. Common pitfalls.
5. Closing insight.

## Structure safeguards

- Do not add examples that were not supplied.
- Do not convert uncertainty into certainty.
- Keep the author's stance intact unless asked to change it.
- If evidence is missing, flag it in change notes rather than fabricating.
